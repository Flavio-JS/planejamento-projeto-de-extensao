# Planejamento do Projeto de Extensão: Website Institucional para ONG

## Fase 1: Imersão e Levantamento de Requisitos (O que fazer?)

O primeiro passo é encontrar a ONG parceira e entender exatamente o que ela precisa.

### 1.1. Identificação do Parceiro

- Selecionar uma ONG, associação comunitária ou abrigo local que não tenha um site, ou cujo site seja muito desatualizado.

- Acordo de Parceria Simples (documento informal de compromisso).

### 1.2. Entrevista e Imersão

- Agendar reuniões com os líderes e voluntários da ONG. Perguntar: 
  - Qual o objetivo principal do site? (Captar doações? Recrutar voluntários? Divulgar eventos?) 
  - Quem é o público-alvo? (Doadores, voluntários, beneficiários?) 
  - Quais informações precisam estar no ar?	

- Documento de Requisitos: Lista de requisitos funcionais e não funcionais (por exemplo: "O sistema deve ter uma página de 'Como Doar'", "O site deve carregar em menos de 3 segundos").

### 1.3. Definição do Escopo do MVP

- Decidir as páginas e funcionalidades mínimas que serão implementadas no projeto de extensão (MVP - Produto Mínimo Viável), dado o prazo da matéria.	

- Escopo Definido: Lista de 4 a 6 páginas/seções essenciais (Ex: Home, Quem Somos, Como Ajudar, Contato).

## Fase 2: Design e Arquitetura (Como será?)

Nesta fase, você transforma os requisitos em um design visual e um plano técnico.

### 2.1. Arquitetura da Informação

- Organizar o conteúdo do site de forma lógica (mapa do site). Definir a hierarquia de páginas e a navegação (menu principal).	

- Diagrama de Navegação/Mapa do Site (fluxograma simples).

### 2.2. Prototipagem de Baixa Fidelidade

- Desenhar esboços a mão ou com ferramentas simples (wireframes) das telas principais para validar o layout e o fluxo com a ONG.	

- Wireframes (Esboços): Desenhos de como serão as páginas principais (onde ficará o menu, o botão de doação, etc.).

### 2.3. Design de Interface (UX/UI) e Alta Fidelidade

- Criar o design visual, cores, tipografia (baseado na identidade da ONG). Usar ferramentas de prototipagem (como Figma ou Adobe XD) para criar as telas finais.	

- Protótipo de Alta Fidelidade (Mockups): Arquivo no Figma ou ferramenta similar com o design final de todas as páginas definidas no MVP.

### 2.4. Definição da Pilha Tecnológica

- Escolher as tecnologias. No 2º período, é ideal focar no Front-end: HTML, CSS e JavaScript.	

- Especificação Técnica Simples: Documento listando as tecnologias, padrões de código e ferramentas que serão utilizadas.

## Fase 3: Desenvolvimento e Implementação (Mão na massa)

Aqui você começa a codificação e a aplicação da responsividade.

### 3.1. Codificação do Estrutura (HTML)

- Montar a estrutura básica de todas as páginas (cabeçalho, rodapé, seções de conteúdo) usando boas práticas de semântica.	

- Páginas HTML Estruturadas (sem estilo visual).

### 3.2. Estilização e Design Responsivo (CSS)

- Aplicar o estilo (cores, fontes) e garantir que o site funcione perfeitamente em telas de celular, tablets e desktops (uso de Media Queries).	

- Arquivos CSS com o design completo e responsividade implementada.

### 3.3. Interatividade (JavaScript - Se houver)

- Implementar funcionalidades básicas (ex: menu hamburger no celular, galeria de fotos simples). 

- Mantenha o JS ao mínimo para este nível.	Páginas e Scripts JS com as funcionalidades básicas.

### 3.4. Otimização para SEO Básico

- Garantir títulos, descrições e URLs amigáveis para ajudar a ONG a ser encontrada em pesquisas do Google.	

- Revisão de Código para inclusão de metatags e otimização de imagens.

## Fase 4: Testes e Validação (Funciona?)

É crucial testar o software e obter o feedback da ONG.

### 4.1. Testes de Funcionalidade

- Verificar se todos os links, botões e formulários (mesmo que protótipos) funcionam como esperado.	

- Checklist de Testes Funcionais (documento com resultados dos testes).

### 4.2. Testes de Usabilidade e Responsividade

- Testar o site em diferentes dispositivos (celular, tablet) e pedir para pessoas não-técnicas da ONG navegarem e darem feedback.	

- Relatório de Usabilidade (listando problemas encontrados e soluções aplicadas).

### 4.3. Validação Final com o Cliente

- Apresentar o site finalizado (ou o protótipo/versão final dentro do escopo) à liderança da ONG para aprovação.	

- Termo de Aceite (formalizando a entrega do projeto/MVP).
